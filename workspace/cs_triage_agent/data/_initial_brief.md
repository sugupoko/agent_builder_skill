# 初期ヒアリング情報 — CS Triage Agent

> このファイルは `/agent-discover` への入力。業務担当者と握る事前情報をまとめる。

---

## 1. お題

B2B 製造業 (部品・工具・FA サプライを大量 SKU で即納する商社) のコールセンター / カスタマーサポート向けに、顧客からの問い合わせメール/チャットを「**トリアージ + 回答ドラフト生成**」するエージェントを構築する。

---

## 2. 重要な前提

- 実コールセンターログは社外秘のため、**合成データ前提**で進める
- ヒアリングは **既に実施済み** — 結果は隣の `hearing_notes.md` に記録
  - CS センター長 富田、ベテランオペ 中島、新人オペ 高橋 の 3 名から取得

---

## 3. このエージェントが扱う問い合わせの典型像

ヒアリングで挙がった代表例:

- 「型番○○の在庫と納期を教えて」
- 「○○の代替品はあるか」
- 「○○と○○の違いは何か (材質 / 耐熱温度 / 精度)」
- 「先週注文した分の配送状況は」
- 「CAD データが欲しい」
- 「請求書を再発行してほしい」
- クレーム系 (「届いたものが違う」「梱包が不十分」)

---

## 4. 設計の核心

1. **LLM には判断させすぎない**: 型番抽出・在庫/価格 DB 引き当て・カテゴリ分類は決定論。LLM は意図補完・回答ドラフト・曖昧質問の深掘りに限定
2. **必ず人間レビュー前提**: オペレーターが最終送信。エージェントは下書き屋
3. **編集者の声 = ベテランオペの判断スキル** を言語化:
   - 「クレーム匂いを早期検知」
   - 「材質質問は耐熱温度や使用環境を逆質問」
   - などをペルソナとして注入

---

## 5. 期待する初期成果物 (`/agent-discover` の出力)

- `spec.md` (初版仕様書)
- `hearing_notes.md` (ヒアリング結果の整理版)
- `usecase_catalog.md` (30 件程度)
- `reports/discover_report.md`
