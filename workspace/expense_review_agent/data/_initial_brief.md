# 初期ヒアリング情報 — Expense Review Agent

> このファイルは `/agent-discover` への入力。業務担当者と握る事前情報をまとめる。

---

## 1. お題

中堅企業（社員 100〜500 名）の経理部に毎月届く **経費 / 請求申請（出張・交際・会議・消耗品・通信・研修等）** を、ルールに照らして自動レビューし「**即承認 / 要修正 / 要人手レビュー / 拒否**」の 4 段階に振り分けるエージェントを構築する。

---

## 2. 重要な前提

- 実申請データは社外秘 + 個人情報含むため、**合成データ前提**で進める
- ヒアリングは **既に実施済み** — 結果は隣の `hearing_notes.md` に記録
  - 経理マネージャ 中村 / ベテラン経理 田島 / 申請者代表 川口（営業部）/ 内部監査 佐々木 / SE 藤本 の 5 名
- 既存システムは **freee / SAP Concur / 楽楽精算** のいずれか想定（具体は本ヒアリングで合意）

---

## 3. このエージェントが扱う申請の典型像

ヒアリングで挙がった代表例:

- 出張交通費（新幹線往復 + 在来線 + タクシー）
- 出張宿泊費（出張規程の上限 1 万円超え判定）
- 接待交際費（顧客との会食、相手先記載必須）
- 会議費（社内会議飲食、人数×単価上限）
- 消耗品費（5,000 円以上は申請書必須）
- 通信費（個人携帯利用分の按分）
- 研修費（書籍 / 外部セミナー、上長承認必要）
- 雑費（用途不明瞭になりやすい）
- **規程違反グレーゾーン**: 一人出張で接待単価超過、社員のみ会食、領収書日付が 60 日以上前、重複申請（同一領収書）

---

## 4. 設計の核心

1. **コードで書けるルールはコードで**: 金額上限・領収書必須・日付期限・科目分類・税区分は決定論。LLM は「グレーゾーンの妥当性判断」「申請者へのフィードバック文言」「過去事例との整合性」に限定
2. **必ず人間レビュー前提**（要レビュー判定の場合）: 経理担当が最終承認。エージェントは「自動承認 OK」「軽微修正で OK」「要人手判断」を仕分け
3. **編集者の声 = ベテラン経理の判断スキル** を言語化:
   - 「金額が中途半端なときは領収書本文を確認」
   - 「同じ店名 + 同じ金額が前月にもないか確認」
   - 「相手先空欄の接待は怪しむ」
   - などをペルソナとして注入
